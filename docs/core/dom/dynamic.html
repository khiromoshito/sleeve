Dynamic Elements ../ Sleeve was originally based on the concept of reactive web programming with state management. It is the framework's building block, and the most important aspect of its architecture.

As a brick to the whole foundation, Sleeve introduces dynamic elements, or elements that can contain direct JavaScript code and update itself whenever triggered.

This solves the problem of displaying dynamic values (values that change) and values dependent from a JS execution. Take a look at the example below, displaying the value of a variable.

Add dynamic as an attribute to make an element dynamic. Then, JavaScript codes can be embedded by enclosing it with double curly brackets: {{ //YourJSCodeHere }} , and it can be placed anywhere within the element (on the tag or within the children), as many as one heart's desire.

The JavaScript Way
Manually, you can instantiate a dynamic element with the Dynamic class, like this one:

var dynamicElement = new DynamicElement(element);

You can then process the element and its script by calling the update method.

var dynamicElement = new DynamicElement(element);
dynamicElement.update();

It's that simple.

But then, the samples above were still static. The element and its dependent values were just updated once (as the element was displayed).

In order to fully grasp the state management capacity of dynamic elements, Sleeve presents a handy object that can handle changes on its value and automatically update dynamic elements dependent to it.

Warning: Dynamic elements inside dynamic elements (or dynamic descendants) are not allowed. Either make the parent dynamic, or not, by specifying which descendants can be dynamic.

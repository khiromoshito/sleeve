Abstract Elements ../ The Abstract Elements provide templating and creating reusable components for web pages, all under a custom HTML tag. This is one step further for semantics.

<something abstract></something>

The code above shows how to create an abstract element something. Simply adding the abstract attribute does the trick, and Sleeve automatically registers something as a new HTML tag.

An instance can then be created just like a normal HTML tag.

<redtext abstract> <p style="color: red"> Hello World </p> </redtext> <redtext></redtext> <redtext></redtext> <redtext></redtext>

Hello World



Warning: In naming abstract elements, only letters, digits (0-9), underscores(_) and dashes (-) are allowed.


With Variables
Abstract elements can have variables. In creating one, just put them between percent signs, like this: %variableName%. They can be present anywhere in the abstract element, and can be of any count.

The instance element can then set values for the variable by adding this attribute:

value-variableName

The value- is constant, so it will be the prefix for each variable setting. This is then followed by the variable name. The example below applies this.

<bluetext abstract> <p style="color: blue; font-size: %fontsize%"> %text% </p> </bluetext> <bluetext value-text="Hello World" value-fontsize="10px"></bluetext> <bluetext value-text="Lorem Ipsum" value-fontsize="20px"></bluetext>

%text%



Warning: In naming variables, only letters, digits (0-9), underscores(_) and dashes (-) are allowed.


Using Models
An instance of abstract elements are formed with a model as the root element of the element. When an abstract element only has one child, then that child becomes the model. Otherwise, if there are no children or there are more than one, then it uses div by default.

Using models can also be identified as extending existing HTML elements. For example, having input as a model element means the abstract element represents an input element.

To configure the model, add the model attribute, set to the specific model element. This code, for example, creates a button abstract element.

<mybutton abstract model="button"> %button-text% </mybutton> <mybutton value-button-text="CLICK ME"></mybutton>
%button-text%


When the model for an abstract element is set, automatically, the children of the abstract element becomes the children of the model element.


Abstracts inside Abstracts
Also, abstract elements can be nested inside other abstract elements, too. The snippet below mixes the two examples above.

<bluetext abstract> <span style="color: blue; font-size: %fontsize%"> %text% </span> </bluetext> <mybluebutton abstract model="button"> <bluetext value-text="%button-text%"></bluetext> </mybluebutton> <mybluebutton value-button-text="CLICK ME" value-fontsize="20px"></mybluebutton>
%text%
